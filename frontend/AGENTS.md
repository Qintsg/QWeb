- `/AGENTS.md`
- `/frontend/AGENTS.md`

## 2. 前端技术事实

技术栈：

- Vue 3
- TypeScript
- Vite
- Vue Router
- Pinia
- Fluent 2 Design System

若仓库已有明确约定，优先沿用：

- 组件组织方式
- API 封装方式
- 路由组织方式
- 状态管理约定
- 主题与 token 约定

若无明确约定，优先保持：

- Composition API
- 强类型
- 明确分层
- 强语义命名
- 可测试的组件结构

---

## 4. 前端代码规范

### 4.1 规范来源
优先遵循：

1. Vue 3 官方规范
2. Fluent 2 官方设计规范
3. TypeScript 官方规范
4. 仓库既有约定
5. 通用工程约定

### 4.2 类型与命名
严格要求：

- 禁止 `var`
- 尽量不使用 `any`
- 优先 `const` / `let`
- 类型定义要有业务语义
- 禁止无语义命名，如 `tmp`、`data`、`obj`、`res`
- 组件名、store 名、API 名必须体现职责

### 4.3 组件与设计系统
- 共享视觉元素优先沉淀为设计系统组件
- 页面层不要反复硬编码颜色、间距、圆角、阴影、状态色
- 导航、表单、反馈、卡片、表格、列表保持 Fluent 2 风格一致
- 不同模块不要出现多套视觉语言

### 4.4 组件边界
- 展示组件与业务组件分离
- 布局组件与页面组件分离
- 表单状态与展示分离
- 风险确认组件与实际业务请求分离
- 文件展示组件与文件权限逻辑分离

### 4.5 未完成实现
默认不保留：

- 空页面
- 假按钮
- 未接线弹窗
- 假数据成功态
- 未处理错误态
- 无权限却仍可点击的危险操作

若确有必要，使用 `TODO:` / `FIXME:` 并说明原因与影响。

### 4.6 注释风格

使用 reStructuredText 风格注释，需要有文件头注释
